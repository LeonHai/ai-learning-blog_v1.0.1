## 2. 核心代码与 AI 审计

```python
users_data = [
    {"id": 1, "name": "Alice", "status": 200},
    {"id": 2, "name": "Bob", "status": 404},
    {"id": 3, "name": "Charlie", "status": 500},
    {"id": 4, "name": "David", "status": None},
    {"id": 5, "name": "Eve", "status": 301}
]

csv_data1 = "name"
csv_data2 = "status"
csv_data3 = {200:"OK", 404:"Not Found", 500:"Server Error"}

for user in users_data:
    if user.get(csv_data2) is None:
        print(user.get(csv_data1,"Unknown User")+":"+"状态码错误")
    else:
        print(user.get(csv_data1,"Unknown User")+":",csv_data3.get(user.get(csv_data2),"Unknown"))
```

### AI 审计建议

* 建议使用变量缓存 `name` 和 `status` 提高可维护性
* 建议使用 f-string 统一输出格式
* 建议使用语义化变量名（FIELD_NAME / STATUS_MAP）
* 异常路径（None）处理方式正确，符合工程规范

# Python 自动化测试学习项目 - AI 导师指导手册

## 欢迎！
本手册旨在指导你如何与你的 AI Python 自动化测试导师（即我）进行高效互动，以确保我严格遵循你设定的教学规则和学习路径。通过本手册，你将能充分利用 AI 的能力，成为一名优秀的自动化测试工程师。

## 1. 核心教学哲学
我们的学习基于以下核心理念：
- **学生为中心**：教学流程完全根据你的需求和反馈进行调整。
- **苏格拉底式引导**：导师不直接给出答案，而是引导你发现和理解。
- **实战导向**：所有知识点和练习都与真实的自动化测试场景紧密结合。
- **防御性编程**：培养你编写健壮、可维护、能处理异常和边界情况的代码。
- **AI 辅助提效**：利用 AI 辅助代码生成与优化，同时训练你的代码审计能力。
- **系统性学习**：先被动学习基础概念，再进行主动实践。

## 2. 教学流程概览 (导师行为准则)
每次学习会话将严格遵循以下步骤：
1.  **温故知新**：导师将提出一个简短的实战问题，回顾上一节课的核心知识点。
2.  **新知识系统讲解**：对于每个新模块或新知识点，导师将：
    *   完整讲解：是什么 (What)、怎么用 (How)、为什么用 (Why)。
    *   提供简单练习：让你被动确认对概念的理解。
    *   明确确认：询问你是否掌握该知识点。
3.  **新知识引入 (针对之前未出现但需讲解的)**：遵循介绍、原理、目的、示例和练习的完整流程。
4.  **场景引入与逻辑拆解**：当基础知识掌握后，导师将引入真实自动化场景，引导你拆解问题。
5.  **代码编写与预期结果**：你将手动编写代码，AI 辅助补全。导师每次提问编码时，必须明确给出**预期的运行结果**。
6.  **防御性检查**：检查代码对异常、边界值的处理能力。
7.  **AI 审计**：对比你的思路与 AI 的优化建议，提升代码审查能力。
8.  **记录与复盘**：更新会话记录 (`session-notes.md`) 和学习进度表 (`progress.md`)。

## 3. 你的职责与互动方式
为了确保教学高效进行，你需要：
- **清晰的意图**：每次开始学习时，明确告知导师你的意图（例如：“开始新一天的学习”、“我们讨论一下 XXX 概念”）。
- **及时反馈**：
    - 如果你对某个知识点感到困惑或觉得太难，**请立即告知导师**。
    - 如果你对导师的教学方式有任何建议或不满，**请立即提出**。
    - 如果你发现导师有任何违反教学规则的行为，**请立即指出**。
- **积极思考与尝试**：即使不确定，也请尝试回答问题或编写代码，你的尝试是学习的关键。
- **文件管理**：定期保存导师提供的 `session-notes.md` 和 `progress.md`。

## 4. 关键文件管理
以下文件是教学过程的核心组成部分，请确保它们始终是最新的，并保存在你的本地项目目录中：

-   **`ai_guide.md`**: 导师的教学行为准则和核心策略。
    *   **用途**：导师每次教学前都会重新加载此文件，以确保行为符合你的要求。
-   **`python-automation-path.md`**: 你的学习路线图和阶段性里程碑。
    *   **用途**：指导导师安排教学内容和难度。
-   **`session-template.md`**: 每日会话记录的模板。
    *   **用途**：导师将按照此模板生成每日学习报告。
-   **`progress.md`**: 你的学习进度仪表盘。
    *   **用途**：记录你的学习进展和各知识点的掌握程度。
-   **`/daily/YYYY-MM-DD/session-notes.md`**: 每日学习会话的详细记录。
    *   **用途**：记录你每日的学习内容、遇到的问题、掌握情况和下一步计划。

## 5. 如何启动/继续学习
1.  **确保所有 `.md` 文件已更新至最新版本。**
2.  **向导师发送指令**：例如：“老师，我们开始新一天的学习吧。”
3.  **享受学习过程！**
